# CV Facts

## Master positioning

George is strongest when positioned as a final-year Computer Engineering student with hands-on applied AI engineering experience across:

- LLM applications and RAG workflows
- Computer vision pipelines and deployment
- Backend APIs for AI systems
- Dockerized ML services and Linux-based development

## Reusable strengths

- Python + SQL is consistent across all target roles
- Strong overlap with FastAPI / Flask / REST APIs
- Strong match for PyTorch / TensorFlow / scikit-learn roles
- Good evidence for LLM, prompt engineering, RAG, LangChain, LlamaIndex, agentic workflows

- Junior AI Engineer:
  - Lead with LLM apps, RAG, prompt engineering, FastAPI, Docker, LangChain, LlamaIndex

## Watchouts

- Avoid overselling years of experience
- Use "hands-on", "applied", "built", "developed", "contributed" rather than implying senior ownership
- Keep CV variant aligned to role family: LLM/applied AI version for Nawy-like jobs, CV version for Synapse-like jobs

## Delivery preferences

- Cover letters should always be exported as PDF deliverables
- Every CV version should stay ATS-friendly:
  - plain single-column layout
  - standard section headings
  - no icons, tables, text boxes, or decorative graphics
  - simple bullet structure and keyword-readable text
- Before finalizing any tailored CV, review the target job description and pull in the most important truthful keywords so the CV matches the role more closely.
