- Use "Final-year Computer Engineering student" rather than "AI/ML Engineer candidate"
- Push Python, SQL, Pandas, NumPy, scikit-learn, TensorFlow, PyTorch near the top

## Siemens SSP AI/ML intern version notes

- Keep final-year student status visible in the first line of the summary.
- Surface Python, C++, TensorFlow, PyTorch, scikit-learn, Pandas, NumPy, Linux, and model-building fundamentals early.
- Reuse project bullets that show AI pipelines, experimentation, and practical engineering discipline rather than purely product copy.
- Mention LLM or agentic interest only briefly after core ML foundations so the CV still reads as balanced AI/ML engineering.
- ATS formatting note from 2026-04-30:
  - Best current Siemens version is a plain single-column one-page LaTeX resume with standard section headers, simple bullets, and no icons, tables, or decorative layout elements.
- Best final structure used on 2026-04-30:
  - Summary -> compact skills block -> Cellula + Elevvo experience -> Shoplifting + Flood API + Student Helper projects -> education and highlights
- Best supporting proof points:
  - Cellula for end-to-end TensorFlow/PyTorch pipelines and production-style inference workflows
  - Elevvo for preprocessing, feature engineering, scaling, and documented evaluation
  - Shoplifting Detection for video understanding and measurable experiment quality
  - Flood Rapid Mapping API for deployable AI services and FastAPI/Docker integration
- Keep teamwork, documentation, and engineering discipline visible because Siemens explicitly values collaboration, planning, and clear experimental reporting.
